# Aesynx v0.35.5 Release Candidate Notes

`v0.35.5` starts the x86_64 QEMU AP startup milestone by making the future AP
startup writer depend on a sealed dispatch authority instead of advisory status
checks.

This release still does not start secondary cores. It proves that shared
bootstrap descriptor tables block AP dispatch-token minting, so later APIC
INIT/SIPI code has to consume an execution-allowed token before it can touch AP
startup resources.

## What Changed

- Added `ApStartupDispatchToken` to `aesynx-core`.
- `ApStartupPreflight::into_dispatch_token` now consumes preflight state and
  mints a dispatch token only when:
  - the caller is the topology owner,
  - every staged AP resource passes preflight validation, and
  - descriptor-table readiness allows AP execution.
- Dispatch tokens are non-`Copy`, non-`Clone`, have private fields, expose only
  bounded inspection helpers, and scrub their resource slots on drop.
- Added `CoreError::StartupPreflightBlocked` so blocked dispatch-token minting
  is explicit rather than inferred from status fields.
- QEMU multicore topology smoke now reports
  `ap_dispatch_token_blocked_ok=true` beside `ap_execution_blocked_ok=true`.
- The generated image manifest records the new dispatch-token marker and names
  the candidate as `v0.35.5`.

## Pentest Fixes

- Added AMD extended CPUID probing for IBRS and IBPB, keeping AMD's independent
  `CPUID.80000008H:EBX` bits separate from Intel's combined leaf-7 bit.
- Split CPU hardening telemetry into `ibpb_supported` and `ibpb_attempted` so
  boot evidence no longer implies a write-only IBPB barrier can be confirmed by
  readback.
- Moved topology epoch mutation behind post-state validation so failed
  invariant checks cannot permanently advance the topology epoch.
- Made AP dispatch tokens explicitly non-`Sync`.
- Rejected AP startup stacks outside the bounded high AP-stack window before
  resources enter the preflight set.
- Tightened startup staging to discovered, unassigned, offline cores only.
- Added explicit timing-safe comparison helpers for AI policy hash/signature
  metadata without reintroducing `PartialEq`.
- Made startup-ticket arrival observation consume the ticket, preventing replay
  from the same ticket value.
- Reordered slab-page reclamation so the class page counter is decremented only
  after the free-list rebuild and page-free transition complete.

## Security Notes

- The future APIC INIT/SIPI path must accept a consuming
  `ApStartupDispatchToken`; it must not re-check advisory status booleans as a
  substitute for authority.
- Shared-bootstrap descriptor tables remain an execution blocker. This release
  intentionally proves the dispatch token cannot be minted in that state.
- Dispatch-token debug output redacts AP startup resources.
- The token is still model authority only. No AP trampoline, APIC write path, or
  secondary-core Rust execution exists yet.

## Non-Claims

- No AP startup trampoline.
- No APIC INIT/SIPI hardware writer yet.
- No secondary core executes Rust code yet.
- No per-core GDT/IDT/TSS/IST installation yet.
- No directed IRQ routing or cross-core message fabric yet.
- No claim that shared-bootstrap descriptor storage is safe for AP execution.
- No per-context-switch IBPB for userspace isolation yet; the current IBPB
  write is boot-time pre-runtime predictor cleanup only.
