# Aesynx v0.16.1 Release Candidate Notes

Status: implementation candidate; pentest and GitHub checks are still required
before tagging.

## Scope

`v0.16.1` closes the first parser and mapper proof gaps identified after the
kernel mapping policy release. It does not add live CR3 activation; that remains
planned for `v0.16.2`. This release adds dependency-free, CI-safe evidence for:

- BootInfo normalization against byte-shaped, bootloader-style fuzz input.
- Named seed coverage for valid maps, empty maps, overlapping maps, adjacent
  maps, overflowing regions, malformed kernel image windows, aarch64 kernel
  windows, and HHDM-present metadata.
- Mapper property sweeps for map/unmap round trips, failed-operation atomicity,
  duplicate physical-frame rejection, bounded range walks, table/index drift,
  and audit detection of raw table corruption.
- A named `cargo xtask fuzz-smoke` gate that runs the BootInfo fuzz smoke and
  mapper property tests without adding external fuzzing dependencies.

The fuzz smoke is intentionally bounded and deterministic. It is not a claim of
full coverage or libFuzzer-style coverage guidance. It is the first repeatable
host fuzz/property gate before normalized boot metadata and mapper state are
used to drive live hardware page tables.

## Deliverables

- `crates/aesynx-boot/src/fuzz.rs`, compiled only for tests.
- Byte cursor and decoder for BootInfo-shaped metadata.
- Named seed corpus for normalization edge cases and redaction-sensitive
  metadata presence.
- Deterministic mutation sweep over 128 byte-shaped cases.
- `crates/aesynx-mm/src/page_table/tests/property.rs`.
- Mapper property tests for single-page and contiguous-range behavior.
- `cargo xtask fuzz-smoke`.
- `scripts/checks.sh` now runs the fuzz/property smoke as part of the standard
  local gate.

## Security Hardening and Fixes Since v0.16.0

- BootInfo normalization now has a bounded fuzz target that exercises
  bootloader-shaped memory-map and metadata inputs without unsafe code or new
  dependencies.
- The named corpus asserts both accepted and rejected normalization paths, so
  the fuzz smoke cannot silently become all-rejecting for curated cases.
- Randomized byte-shaped cases must always resolve to either a valid normalized
  `BootInfo` or an explicit `BootInfoError`; the target must not panic.
- Accepted fuzz cases re-check summary accounting and kernel-image sanity after
  normalization.
- Mapper property tests repeatedly prove that map/unmap returns the mapper to
  its empty state across a page window.
- Failed single-page operations, duplicate physical-frame maps, oversized range
  walks, accounting drift, stale table links, and malformed next-table links
  are checked for fail-closed behavior and non-mutation.
- The repository-wide `scripts/checks.sh` gate now includes the new bounded
  fuzz/property smoke.
- x86_64 interrupt-state detection no longer marks stack-touching
  `pushfq`/`pop` assembly as `nomem`.
- Runtime IDT interrupt-gate installation now refuses to mutate 16-byte IDT
  entries after maskable interrupts have been enabled, avoiding a torn-entry
  update window in the current early-boot model.
- Limine BootInfo normalization is now explicitly one-shot, so private Limine
  request statics cannot be normalized repeatedly in future multi-entry paths.
- Descriptor and exception setup status reports now expose whether this call
  performed initialization, making accidental re-entry visible instead of a
  silent no-op.
- Timer smoke now idles with the architecture wait-for-interrupt primitive
  instead of a pure spin loop while waiting for PIT IRQ0 ticks.
- The raw x86_64 exception-frame ABI now has compile-time size and field-offset
  assertions tied to the assembly stubs.
- Style-only unchecked arithmetic in allocator and mapper status paths was
  replaced with checked or saturating arithmetic.
- Firmware-provided framebuffer and RSDP addresses remain metadata-only at this
  release. Typed provenance wrappers are tracked as required before any
  framebuffer or ACPI consumer can dereference or write through those pointers.
- aarch64 `wait_for_interrupt` now uses the architectural `wfi` instruction on
  aarch64 targets, mirroring the x86_64 `hlt` idle contract while preserving
  host-test fallback behavior.
- BootInfo fuzz seeds now assert exact `BootInfoError` variants for every
  normalizer error class and include deterministic metadata payload bytes for
  framebuffer, RSDP, device-tree, and HHDM cursor paths.
- The deterministic fuzz mutation sweep now checks repeatability of accept/error
  outcomes instead of relying on a vacuous outcome-shape assertion.
- Device resume and IPC request documentation now mark forward-looking trust
  boundaries for stopped-device resume and destination-core validation.
- IPC inline payload construction now uses a checked `u8` length conversion.
- Mapper property tests now cover the exact 32-page bounded-frame-index success
  boundary and split the over-capacity rejection into a separate test.

## Non-Claims

- No live CR3 activation.
- No production page-table ownership.
- No coverage-guided fuzzing engine yet.
- No formal Kani/CBMC proof yet.
- No typed framebuffer or ACPI/RSDP pointer dereference API yet.
- No userspace, syscall, or hostile ring-3 boundary yet.
