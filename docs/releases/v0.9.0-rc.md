# Aesynx v0.9.0 Release Candidate Notes

Status: implementation candidate; not tag-ready until pentest and GitHub checks
are complete.

## Scope

`v0.9.0` improves the early x86_64 exception diagnostics added in v0.8. It
keeps the same GDT/TSS/IDT setup and extends the page-fault path so memory-work
failures expose useful bounded serial diagnostics.

It does not claim page-table ownership, page-fault recovery, CR2-driven memory
management, interrupt-controller setup, userspace transitions, or complete
unhandled-vector diagnostics.

## Deliverables

- Expanded interrupt frame copy for vector, error code, RIP presence/offset,
  CS, and RFLAGS.
- Page-fault error-code decoder for present, write, user, reserved-bit,
  instruction-fetch, protection-key, shadow-stack, and SGX bits.
- CR2 fault-address capture for page-fault diagnostics.
- CR3 low-bit summary capture for page-fault diagnostics.
- Public RFLAGS summary and interrupt-enable state output.
- QEMU exception-smoke gate now requires fault address, CR3, RFLAGS,
  interrupt-state, and decoded page-fault markers.
- v0.9 QEMU image, manifest, and serial-log names.

Expected exception-smoke page-fault diagnostic shape:

```text
exception vector=page-fault error=0x<n> rip_present=<bool> rip_offset=0x<n> cs=0x<n> frame_rflags=0x<n> cr2=0x<n> cr3_offset=0x<n> rflags=0x<n> interrupts_enabled=<bool> present=<bool> write=<bool> user=<bool> reserved=<bool> instruction=<bool> protection_key=<bool> shadow_stack=<bool> sgx=<bool>
[TEST] pagefault=ok
[TEST] exception=ok
```

## Security Notes

- The CR2 fault address is printed only on the early exception diagnostic path.
  This is intentional for v0.9 memory debugging and should be revisited before
  exposing page-fault telemetry to untrusted userspace or persistent logs.
- CR3 remains redacted to low flag/PCID bits as `cr3_offset`.
- RIP remains redacted to presence and low page offset rather than a raw kernel
  text address.
- The returning breakpoint stub remains register-preserving and stack-aligned
  before calling Rust dispatch.
- Page-fault and double-fault paths still halt after diagnostics.

## Verification

Required before pentest handoff:

```bash
scripts/checks.sh
scripts/generate-sbom.sh
cargo xtask build-kernel
cargo xtask image
cargo xtask qemu
cargo xtask qemu --panic-smoke
cargo xtask qemu --exception-smoke
```

## Known Limitations

- No CR2-based recovery exists yet.
- No page-table walk or page-permission lookup exists yet.
- No general interrupt-controller setup exists yet.
- No user/kernel privilege transition exists yet.
- No SMP/per-core IDT or TSS exists yet.
