# Aesynx v0.6.0 Release Candidate Notes

Status: implementation candidate; not tag-ready until pentest and GitHub checks
are complete.

## Scope

`v0.6.0` is the early-diagnostics milestone. It keeps the v0.5 Limine BootInfo
path and adds readable panic output plus a machine-checked deliberate panic
smoke.

It does not claim interrupt handling, exception handling, page-fault decoding,
raw register dumps, SMP-safe diagnostics, allocator-backed logging, or
persistent telemetry buffers.

## Deliverables

- Safe boot phase tracking in `aesynx-kernel::diagnostics`.
- No-alloc `DiagnosticRecord` formatting with core, phase, component, log
  level, and bounded single-record messages.
- Validated diagnostic component names to prevent bracket or record injection.
- Panic output with core, phase, file, line, column, and message.
- Redacted x86_64 early register summary for `rsp`, `rbp`, `rflags`, and `cr3`.
- Opt-in `panic-smoke` kernel feature.
- `cargo xtask qemu --panic-smoke` QEMU serial expectation, including the panic
  register-summary line.
- Local and GitHub gates cover the panic-smoke feature and QEMU smoke.
- v0.6 QEMU image and manifest names.
- Split xtask image tests to preserve the modularity file-size rule.

Expected normal serial:

```text
[core=0][phase=entry][kernel][INFO] serial initialized
[core=0][phase=bootinfo-normalized][kernel][INFO] bootinfo normalized
Aesynx: booting
arch=x86_64 platform=qemu
memmap regions=<n> usable=<n> usable_bytes=<n>
rsdp=present
[TEST] bootinfo=ok
[TEST] boot=ok
```

Expected panic-smoke serial:

```text
Aesynx: panic during early boot
[core=0][phase=panic-smoke][kernel][FATAL] panic handler entered
panic core=0 phase=panic-smoke
panic location=crates/aesynx-kernel/src/main.rs line=<n> column=<n>
panic message=intentional v0.6.0 panic smoke
panic registers=rsp_present=<bool> rbp_present=<bool> rsp_align=<n> rbp_align=<n> rflags=0x<n> cr3_offset=0x<n>
[TEST] panic=ok
```

## Security Notes

- Diagnostics are implemented in safe Rust.
- Structured diagnostic messages use `LogMessage`, so record separators are
  rejected before formatting.
- Structured diagnostic components reject empty, overlong, uppercase, bracketed,
  or newline-containing names before formatting.
- Boot phase state is a single bounded atomic byte with invalid-value fallback.
- The panic-smoke feature is opt-in and not enabled for normal boot images.
- Panic output reports source location, phase, and redacted register summary
  without exposing KASLR-sensitive boot addresses.

## Verification

Required before pentest handoff:

```bash
scripts/checks.sh
scripts/generate-sbom.sh
cargo xtask build-kernel
cargo xtask image
cargo xtask qemu
cargo xtask qemu --panic-smoke
```

## Known Limitations

- Raw register dumps are intentionally not emitted.
- Core ID is the early single-core boot identity.
- No exception handlers are installed yet.
- No page-fault smoke exists yet.
- No persistent event ring exists yet.
