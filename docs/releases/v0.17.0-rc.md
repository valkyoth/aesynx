# Aesynx v0.17.0 Release Candidate Notes

Status: implementation candidate; not tag-ready until pentest and GitHub
checks are green.

## Scope

`v0.17.0` enables the first bounded kernel heap after the v0.16 memory
enforcement gates. The release intentionally uses a simple static bump
allocator so `alloc` can be proven in QEMU before v0.18 replaces it with a
slab/page-backed heap for long-lived kernel data.

## Delivered So Far

- The normal kernel image now links `alloc` and installs a kernel-local global
  allocator for the non-diagnostic boot path.
- `crates/aesynx-kernel/src/early_heap.rs` provides a 64 KiB page-aligned
  static bump heap in kernel BSS.
- The allocator initializes only after the Aesynx-owned CR3 root is active and
  post-CR3 CPU hardening has passed.
- QEMU normal boot now smokes `Box`, `Vec`, and `BTreeMap` allocations.
- The heap smoke uses fallible reservation to prove oversized allocations fail
  closed instead of silently succeeding.
- Host tests cover pre-initialization rejection, one-shot initialization,
  aligned nonoverlapping range allocation, and oversized request rejection
  without advancing the bump pointer.
- The boot-smoke contract now requires `heap bytes=...` and `[TEST] heap=ok`
  and rejects `[TEST] heap=fail`.
- QEMU image names and manifests now use the `v0.17.0` release-candidate
  version.
- Xtask tests now pin the v0.17.0 boot, panic, exception, and timer artifact
  names plus the manifest release title.

## Verification

- `cargo check -p aesynx-kernel --target x86_64-unknown-none`.
- `cargo test -p aesynx-kernel early_heap`.
- `cargo xtask qemu`.
- `cargo xtask qemu-suite`.
- `scripts/checks.sh`.

## Non-Claims

- No general-purpose free/reuse semantics; `dealloc` is intentionally a no-op
  for the v0.17 bump allocator.
- No slab classes or page-backed large allocations yet.
- No heap compaction, quarantine, leak reporting, or per-subsystem accounting.
- No allocation while holding locks policy yet; concurrency discipline remains
  a later SMP prerequisite.
- No user allocator, process heap, or userspace ABI.
