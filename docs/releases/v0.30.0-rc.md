# Aesynx v0.30.0 Release Candidate Notes

`v0.30.0` adds the first versioned telemetry event schema. The goal is to make
kernel facts portable across boot output, future trace export, and the later OS
world model without exposing raw authority-bearing identifiers by default.

## What Changed

- Added `TELEMETRY_SCHEMA_VERSION = 1`.
- Added stable `TelemetryEventId` values for boot-phase, capability-fault, and
  scheduler-decision events.
- Added `TelemetryEventHeader` with schema version, event ID, sequence, and
  typed core ID.
- Added typed payloads for boot phases, capability faults, and scheduler
  decisions.
- Added `PerCoreEventRing`, a fixed-capacity no_std per-core event buffer with
  checked capacity, monotonic event sequence numbers, ordered lookup, and
  aggregate summaries.
- Added scheduler-event debug redaction so selected task identity remains
  available to typed kernel code but is not printed by event `Debug`.
- Added a kernel telemetry-events smoke that records boot, capability, and
  scheduler events, validates schema/version/order/count invariants, and prints
  a serial summary before `[TEST] telemetry-events=ok`.
- Updated QEMU image names/manifests to `v0.30.0` and made the boot smoke
  require `telemetry-events schema=1 events=` plus `[TEST] telemetry-events=ok`.

## Security And Correctness

- Event IDs are explicit numeric schema values, not derived from enum layout in
  consumers.
- Event headers are kernel-stamped by the ring; callers supply only typed
  payloads.
- The event ring rejects zero-capacity and full buffers without mutating state.
- Event sequence counters fail closed on overflow.
- Scheduler event debug output redacts selected task identity. Core IDs remain
  visible as local non-secret scheduling context until SMP tenancy becomes a
  boundary.
- The schema is a local in-memory boot/telemetry format only. It is not yet a
  stable userspace ABI, persistent log format, IPC export format, or trace file
  format.

- `cargo xtask qemu-suite`.

## Known Limitations

- The per-core event ring is local and unsynchronized. SMP-safe publication,
  IRQ-safe producers, and cross-core merge ordering remain future gates.
- There is no trace export tool yet; v0.31.0 owns offline decoding.
- There is no stable external serialization format yet. Future serializers must
  reuse explicit redaction adapters and must not bypass redacted debug policy.
- Boot-phase event values intentionally mirror the current kernel diagnostic
  phases but are not wired to every phase transition yet.
