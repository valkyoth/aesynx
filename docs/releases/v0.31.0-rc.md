# Aesynx v0.31.0 Release Candidate Notes

`v0.31.0` makes the first telemetry event schema useful outside the kernel by
adding serial `trace-event` records and an offline decoder.

## What Changed

- Added `tools/trace-decode`, a dependency-free host tool that decodes schema-v1
  serial trace-event lines into line-based offline analysis output.
- Added `cargo xtask trace-decode <serial-log>` as the repository command path
  for the decoder.
- Added kernel serial `trace-event` lines for boot-phase, capability-fault, and
  scheduler-decision events in the telemetry event smoke.
- Updated QEMU boot smoke markers and image names to `v0.31.0`.
- Documented the schema-v1 trace format in `docs/telemetry-event-schema.md`.

## Security And Correctness

- The decoder accepts only `schema=1`.
- Unknown event kinds, unknown fields, duplicate fields, invalid numeric fields,
  and missing trace events fail closed.
- Core IDs remain visible under the central `VisibleLocalCoreId` export policy;
  they are still treated as local non-secret scheduling context until SMP
  tenancy becomes a boundary.
- Scheduler selected-task identity must already be `selected_task=<redacted>`.
  The decoder rejects scheduler events that carry a raw task ID.
- The exported line format is offline boot-analysis output only. It is not a
  stable userspace ABI, persistent log format, IPC export format, or coherent
  cross-core ordering contract.

## Known Limitations

- The decoder is line-based; JSON output remains a future extension.
- Trace input currently comes from QEMU serial output, not a persistent kernel
  trace buffer.
- Cross-core merge ordering, IRQ-safe producers, per-core trace publication,
  and stable userspace trace subscriptions remain future gates.

- Run the normal repository gates.
- Run `cargo xtask qemu-suite`.
- Run `cargo xtask trace-decode build/qemu/aesynx-v0.31.0.serial.log`.
- Generate/update SBOM evidence.
- Complete the pentest handoff and store final evidence in
  `security/pentest/v0.31.0.md`.
- Run `cargo xtask release-ready v0.31.0` after the pentest report, SBOM, and
  exact tag commit are ready.
