# Aesynx v0.12.0 Release Candidate Notes

Status: tag-ready; pentest and GitHub checks are complete. The final pentest
report names the exact tag commit.

## Scope

`v0.12.0` turns the controlled QEMU timer ticks from `v0.11.0` into the first
kernel time model. The release adds checked tick-rate conversion, monotonic
instants, time spans, bounded sleep requests, timeout checks, and a timer-smoke
delayed log event that wakes from the sleep queue.

This is still an early smoke path. It proves the model and serial contract in
QEMU, but it does not claim a calibrated production clock, APIC timer,
TSC-deadline timer, scheduler-integrated sleep service, preemption, driver IRQ
routing, or SMP timer distribution.

## Deliverables

- `aesynx-time::TickRate` with zero-rate rejection and checked tick-to-nanosecond
  conversion.
- `aesynx-time::MonotonicInstant` and `TimeSpan` as allocation-free typed time
  values.
- `aesynx-time::SleepQueue` with bounded fixed-capacity storage and due-wake
  selection.
- `aesynx-time::SleepRequest` with task ID, monotonic deadline, and wake ID.
- `aesynx-time::Timeout` with monotonic expiry checks.
- Kernel timer smoke setup records the configured PIT rate and schedules a
  delayed sleep request.
- The timer callback model is represented as a due wake event; arbitrary
  executable callbacks from IRQ context are deliberately deferred.
- QEMU timer smoke now requires `timer delayed-log` and `[TEST] sleep=ok` in
  addition to the three tick markers and final timer marker.
- Image, manifest, serial-log names, xtask help text, and smoke marker tests
  track the v0.12 release contract.

Expected timer-smoke serial shape:

```text
timer setup=pit irq=0 vector=0x20 target_ticks=3 hz=100
timer tick 1
timer tick 2
timer delayed-log task=0 wake_id=1 at_ns=<n> ticks=<n>
[TEST] sleep=ok
timer tick 3
[TEST] timer=ok
```

## Security Notes

- Tick-rate construction rejects zero before conversion is possible.
- The QEMU smoke clock rate comes from the x86_64 timer status that programmed
  the PIT, avoiding an independent kernel-side magic number.
- Tick-to-nanosecond conversion uses checked arithmetic and fails closed on
  overflow.
- Sleep requests are stored in a fixed-size queue with no allocator dependency.
- Full sleep queues return `QueueFull` rather than overwriting existing requests.
- The due-wake path returns only requests whose monotonic deadline has passed
  and chooses the earliest due deadline before slot order.
- Monotonic conversion errors in timer smoke are emitted as serial diagnostics
  rather than being silently ignored.
- Timer work is delivered as an explicit wake event rather than a function
  pointer callback from IRQ context.
- Timer smoke still enables external interrupts only for the controlled QEMU
  path after GDT, IDT, and interrupt-controller setup.
- Normal boot still does not enable external interrupts.
- Kernel builds still disable SSE/AVX until explicit FPU/SIMD context ownership
  exists.

## Verification

Required before pentest handoff:

```bash
scripts/checks.sh
scripts/generate-sbom.sh
cargo xtask build-kernel
cargo xtask image
cargo xtask qemu
cargo xtask qemu --panic-smoke
cargo xtask qemu --exception-smoke
cargo xtask qemu --timer-smoke
```

## Known Limitations

- PIT is a QEMU smoke path and compatibility timer, not the long-term timer
  backend.
- No APIC MMIO register writes exist yet.
- No TSC-deadline timer path exists yet.
- No calibrated production clock source exists yet.
- No scheduler-integrated sleep service exists yet.
- No scheduler preemption exists yet.
- No production external IRQ routing to drivers or services exists yet.
- No SMP or per-core timer state exists yet.
