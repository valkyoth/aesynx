# Aesynx v0.16.2 Release Candidate Notes

Status: implementation candidate; not tag-ready until the kernel-owned
address-space activation work is complete, pentest is green, and GitHub checks
are green.

## Scope

`v0.16.2` turns the verified v0.16 mapping policy toward real x86_64 page
tables. The first implementation slice exports audited mapper state into a
hardware-shaped page-table image with physical next-table entries, redacted
debug output, and QEMU-visible smoke evidence.

This release is not tag-ready yet. The final `v0.16.2` target remains a
surviving CR3 switch to an Aesynx-owned root table.

## Delivered So Far

- `aesynx-mm` can export an audited `PageTableMapper` into an
  `X86_64PageTableImage`.
- Hardware-image export validates the physical root-table arena, rewrites
  model next-table indices into physical next-table entries, and preserves leaf
  entries.
- Intermediate hardware entries derive writable/user bits from their child
  subtree instead of blindly granting every parent entry maximum authority.
- Hardware-image debug output redacts physical addresses and raw table entries.
- Kernel mapping smoke now exports the policy-derived mapper to a hardware
  image and requires `hardware_image_ok=true` before
  `[TEST] paging-policy-model=ok`.
- QEMU smoke metadata now records the `hardware_image_ok=true` marker.

## Remaining For Tag

- Allocate and zero the real page-table arena through the checked frame
  allocator path instead of using a smoke-local physical root constant.
- Copy the hardware image into the allocated page-table frames.
- Add the x86_64 CR3 load primitive with a tight unsafe boundary.
- Re-run policy verification against the table image immediately before CR3
  load.
- Switch CR3 to the Aesynx-owned root and verify execution continues.
- Keep page-fault and timer smokes green after the switch.
- Add `[TEST] kernel-cr3=ok` only when CR3 is actually switched.

## Verification

- `cargo test -p aesynx-mm hardware_image`.
- `cargo test -p aesynx-kernel kernel_mapping_policy`.
- `cargo test -p xtask qemu_markers_track_v0_16_contracts`.
- `cargo xtask qemu-suite`.
- `scripts/checks.sh`.
- Pentest follow-up before tag.
- GitHub CI green before tag.

## Non-Claims

- No active CR3 replacement yet.
- No production page-table frame allocation yet.
- No full process isolation or userspace enforcement yet.
- No kernel stack guard or CPU hardening-bit enablement yet.
