# Aesynx v0.14.0 Release Candidate Notes

Status: implementation candidate; pending pentest and GitHub checks before tag.

## Scope

`v0.14.0` adds the first safe bitmap frame allocator model in `aesynx-mm` and a
normal-boot QEMU smoke that seeds a bounded allocator window from the checked
Limine memory map. The smoke verifies one-frame allocation/free, contiguous
allocation/free, debug state, double-free detection, and atomic failure paths
before emitting `[TEST] frame-allocator=ok`.

This is not a production global physical-memory allocator yet. It does not
claim page-table ownership, heap allocation, bootloader memory reclamation,
per-core frame caches, full-system frame ownership, DMA/IOMMU ownership, or
runtime reclamation.

## Deliverables

- `aesynx-mm::BitmapFrameAllocator`.
- `FrameRegionKind` and `FrameState` debug model.
- Checked allocator construction with capacity and address-overflow rejection.
- Full-frame region seeding using 4 KiB frame boundaries.
- Region-overlap rejection during allocator seeding.
- One-frame allocation and free.
- Contiguous frame allocation and free.
- Validate-then-commit behavior for region seeding and contiguous frees, so
  failed calls do not partially mutate allocator state.
- Double-free detection.
- Reserved/kernel/bootloader/device/ACPI/bad debug states.
- QEMU normal-boot smoke for a bounded usable boot-map window.
- Normal QEMU boot serial markers for frame allocator status and
  `[TEST] frame-allocator=ok`.
- v0.14 QEMU image, manifest, serial-log names, help text, and marker tests.

Expected normal boot serial shape:

```text
memory total_bytes=<n> total_frames=<n> regions=<n>
memory usable_bytes=<n> usable_frames=<n> usable_regions=<n>
memory reserved_bytes=<n> reserved_frames=<n> reserved_regions=<n> kernel_bytes=<n> bootloader_bytes=<n> framebuffer_bytes=<n> acpi_bytes=<n> bad_bytes=<n>
[TEST] memory-map=ok
frame-allocator total_frames=<n> known_frames=<n> free_before=<n> free_after=<n> reserved_frames=<n> contiguous_count=2 double_free_check=true
[TEST] frame-allocator=ok
[TEST] bootinfo=ok
[TEST] boot=ok
```

## Security Notes

- Allocator seeding uses full frames contained in each boot memory region.
- Allocator construction rejects zero-frame windows, capacity overflow, capacity
  undersizing, and base-plus-count address overflow.
- Region seeding and contiguous frees validate every touched frame before
  committing bitmap mutations, preventing partial state commits on error paths.
- Allocation is possible only from frames marked `Free`.
- Reserved, kernel, bootloader, device, ACPI, and bad-memory frames are not
  returned by allocation.
- Freeing an already-free frame returns `DoubleFree`.
- Freeing reserved or unknown frames fails closed.
- Normal boot prints frame counts and status only; it does not print raw frame
  numbers or physical addresses.

## Known Limitations

- The QEMU smoke uses a bounded early allocator window, not full-system frame
  ownership.
- No page-table mapper exists yet.
- No heap allocator exists yet.
- No bootloader memory reclamation exists yet.
- No per-core frame cache exists yet.
- No frame ownership transfer into object memory exists yet.
- No DMA/IOMMU frame ownership exists yet.
