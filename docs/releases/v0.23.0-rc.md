# Aesynx v0.23.0 Release Candidate Notes

Status: release candidate

## Summary

`v0.23.0` starts Phase 6 by adding the host-side object graph model that will
guide the kernel object registry in `v0.24.0`. The model defines checked object
identity, explicit object kinds, immutable nodes, append-only graph insertion,
and reachability rules before object ownership becomes live kernel state.

## Added

- Added `models/aesynx-object-model` as a workspace member.
- Added `ModelObjectId`, a nonzero wrapper around `aesynx_abi::ObjectId` with
  redacted `Debug` output.
- Added `ObjectKind` for the first native object classes: memory, endpoint,
  queue, task, address space, device, driver, package, snapshot root,
  persistent node, telemetry stream, and world fact.
- Added `ImmutableNode` with explicit kind, optional predecessor, and immutable
  object references.
- Added `ObjectGraph` with duplicate-ID rejection, missing-reference rejection,
  and reachability over references plus predecessor links.

## Security And Correctness

- Object ID zero is rejected by the host model so a null-like object identity
  cannot become a valid graph root by accident.
- Object ID debug output is redacted to avoid training future diagnostics to
  leak authority-bearing identifiers.
- Immutable nodes reject self-references and duplicate references before they
  can enter a graph.
- Graph insertion is append-only and rejects references to objects that are not
  already present, keeping the model deterministic for later kernel registry
  work.
- Reachability fails closed if a referenced object is missing.
- The v0.23.0 pentest follow-up documents and asserts the freestanding
  interrupt-masking contract for future `set_ring0_stack()` callers before the
  global TSS `rsp0` field is updated.
- Limine framebuffer and RSDP pointer fields now use volatile reads for
  consistency with the rest of the bootloader-owned handoff boundary.
- The x86_64 timestamp unsafe block now keeps the non-serializing `rdtsc`
  caveat inside the `SAFETY:` comment auditors grep for.

## Known Limitations

- This is a host model only. It does not create kernel objects, expose a kernel
  object registry, bind object IDs to capabilities, persist objects to disk, or
  implement snapshot rollback yet.
- Reachability uses in-memory host collections and is not the final kernel data
  structure.
- The QEMU serial marker set is unchanged from `v0.22.0`; this release adds
  host-model evidence for the next kernel object milestone.
