# Aesynx v0.10.0 Release Candidate Notes

Status: tag-ready; pentest and GitHub checks are complete.

## Scope

`v0.10.0` starts the timer/time phase by preparing hardware interrupt handling
without enabling periodic interrupts yet. The x86_64 boot path now remaps the
legacy 8259 PIC to vectors `0x20..0x2f`, masks it, detects whether CPUID reports
a local APIC, records that local APIC MMIO activation is deferred until memory
mapping exists, and exposes a checked IRQ-to-vector policy.

It does not claim APIC MMIO programming, timer ticks, IRQ delivery to drivers,
SMP interrupt routing, or userspace IRQ endpoints.

## Deliverables

- `aesynx-arch-x86_64::interrupts` module for interrupt-controller baseline
  state.
- Legacy PIC remap and mask-all path over admitted 8259 PIC I/O ports.
- Local APIC presence detection through CPUID.
- Explicit `DeferredUntilMmioMapping` APIC mode when APIC hardware is present.
- Checked IRQ vector allocation for legacy IRQ lines 0 through 15 at vector
  base `0x20`.
- Legacy PIC EOI path for valid IRQ lines, including spurious IRQ7/IRQ15 ISR
  checks.
- `InterruptController` trait implementation for the x86_64 baseline.
- Boot diagnostic line and `[TEST] irq=ok` QEMU marker.
- Bytecode verifier offset checks now validate the full 8-byte `u64` access
  width before producing `VerifiedOffset`.
- Serial write helpers expose whether transmit timeouts dropped output.
- v0.10 QEMU image, manifest, and serial-log names.

Expected boot diagnostic shape:

```text
interrupt setup=baseline legacy_pic_masked=<bool> local_apic_present=<bool> local_apic_mode=<mode> irq_vector_base=0x20 irq_vector_count=16
[TEST] irq=ok
```

## Security Notes

- The PIC I/O boundary is limited to four admitted 8259 ports and the existing
  COM1 UART ports.
- The legacy PIC is remapped before masking so spurious PIC interrupts cannot
  arrive in the CPU exception-vector range.
- Spurious IRQ7 and IRQ15 acknowledgement checks the PIC ISR before deciding
  whether to send EOI.
- `enable_irq` and `send_ipi` intentionally return
  `ControllerUnavailable`; APIC MMIO setup is not guessed before page-table and
  MMIO ownership exist.
- IRQ vectors are checked values, rejecting lines outside the legacy 0-15
  range.
- The boot path remaps and masks legacy PIC IRQs before any future interrupt
  enablement.
- The local APIC path is detection-only in this release.

## Known Limitations

- No APIC MMIO register writes exist yet.
- No periodic timer interrupt exists yet.
- No external interrupt handler stubs are installed yet.
- No IRQ routing to services or drivers exists yet.
- No SMP or per-core interrupt-controller state exists yet.
