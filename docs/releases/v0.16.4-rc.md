# Aesynx v0.16.4 Release Candidate Notes

Status: implementation candidate; not tag-ready until pentest and GitHub
checks are green.

## Scope

`v0.16.4` removes the temporary modularity exception carried after the v0.16.3
pentest follow-up. It is intentionally a structure-preserving Limine handoff
split, not a boot protocol change.

## Delivered So Far

- `crates/aesynx-kernel/src/limine.rs` now keeps the normalization flow,
  pointer validation, payload validation, one-shot normalization claim, and
  BootInfo construction.
- `crates/aesynx-kernel/src/limine/abi.rs` now owns Limine protocol structs,
  magic constants, memory-map kind constants, request statics, link-section
  markers, and ABI layout assertions.
- The private ABI module now compile-time asserts the size and consumed field
  offsets for every transcribed Limine request/response structure used by the
  handoff path.
- The `limine.rs` temporary modularity exception has been removed from
  `docs/modularity-policy.md`.
- `docs/unsafe-policy.md`, `docs/security-controls.md`, and
  `scripts/validate-security-policy.sh` now track the split unsafe boundary.
- QEMU image names and manifests now use the `v0.16.4` release-candidate
  version.
- Xtask tests now pin the v0.16.4 boot, panic, exception, and timer artifact
  names plus the manifest release title.

## Verification

- `scripts/validate-modularity-policy.sh`.
- `scripts/validate-security-policy.sh`.
- `cargo check -p aesynx-kernel --bins`.
- `cargo xtask qemu-suite`.
- `scripts/checks.sh`.

## Non-Claims

- No new Limine requests or bootloader protocol behavior.
- No bootloader memory reclamation.
- No SMP topology parsing.
- No framebuffer writer.
- No change to the v0.16.3 CPU hardening or stack guard claims.
