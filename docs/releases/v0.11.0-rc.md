# Aesynx v0.11.0 Release Candidate Notes

Status: implementation candidate; pending pentest and GitHub checks before tag.

## Scope

`v0.11.0` adds the first controlled timer interrupt path. The release uses the
legacy PIT as the chosen QEMU timer source, routes IRQ0 through the already
remapped vector `0x20`, records a tick counter, prints three bounded tick
markers, acknowledges each interrupt, disables IRQ0 after the fixed target, and
then halts.

This is a smoke-proven timer foundation, not yet a production clock service. It
does not claim APIC timer programming, TSC-deadline timers, calibrated monotonic
time, sleep queues, scheduler preemption, driver IRQ routing, or SMP timer
routing.

## Deliverables

- `aesynx-arch-x86_64::timer` module for the PIT-backed QEMU timer smoke path.
- IRQ0 handler stub with full general-purpose register save/restore and ABI
  stack alignment before calling Rust dispatch.
- Checked IDT interrupt-gate installer for vectors `0x20..`.
- PIT channel 0 and command-port admission through the existing typed port
  boundary.
- Interrupt-enable, interrupt-disable, interrupt-state, and timestamp CPU
  helpers for the architecture trait.
- Opt-in kernel `timer-smoke` feature and `BootPhase::TimerSmoke`.
- `aesynx-time::TickCounter` for monotonic tick recording with overflow checks.
- `cargo xtask qemu --timer-smoke` image, manifest, serial-log names, and
  serial marker contract.
- `scripts/checks.sh` clippy gate for the `timer-smoke` kernel feature.

Expected timer-smoke serial shape:

```text
timer setup=pit irq=0 vector=0x20 target_ticks=3
timer tick 1
timer tick 2
timer tick 3
[TEST] timer=ok
```

## Security Notes

- Normal boot does not enable external interrupts in this release.
- The timer smoke path enables interrupts only after GDT, IDT, and interrupt
  controller baseline setup has completed.
- IRQ enablement is limited to supported legacy IRQ0; other legacy IRQ lines
  still reject enablement until real routing exists.
- PIT access is admitted only for ports `0x40` and `0x43`.
- The IRQ0 stub saves registers before Rust dispatch, preserves exact unwind
  state across stack alignment, restores registers before `iretq`, and does not
  return to Rust after the interrupt frame is consumed.
- The dispatch path sends EOI through the checked interrupt-controller path and
  masks IRQ0 once the fixed three-tick target is reached.
- Timer output is bounded to deterministic smoke markers.


## Known Limitations

- PIT is a QEMU smoke path and compatibility timer, not the long-term timer
  backend.
- No APIC MMIO register writes exist yet.
- No TSC-deadline timer path exists yet.
- No calibrated monotonic clock conversion exists yet.
- No sleep queue or scheduler preemption exists yet.
- No production external IRQ routing to drivers or services exists yet.
- No SMP or per-core timer state exists yet.
