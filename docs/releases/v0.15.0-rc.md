# Aesynx v0.15.0 Release Candidate Notes

Status: implementation candidate; pending pentest and GitHub checks before tag.

## Scope

`v0.15.0` adds the first safe bounded page-table mapper model in `aesynx-mm`
and a normal-boot QEMU smoke that exercises map, contiguous range
map/protect/unmap, permission lookup, page-presence checks, contiguous range
lookup, mapped-range checks, permission change, unmapped range checks,
kernel-range policy checks, user-range policy checks, kernel-only mapping
policy checks, high-half kernel user-access guard checks, low-half user
kernel-privilege guard checks, kernel/user address-space candidate preflights,
read-only mapping visit, write-protected range policy checks,
non-executable range policy checks, executable range policy checks,
normal-memory range policy checks, local range policy checks, no-executable
address-space policy checks, no-writable address-space policy checks, no-device
address-space policy checks, no-global address-space policy checks, redacted
mapping summaries, virtual range permission verification, translated-offset
lookup, unmap with empty-table reclamation, explicit TLB flush targets, and
conservative TLB flush merging. The mapper is x86_64-shaped and uses four
512-entry levels, but it does not
activate a new hardware CR3 root yet.

This is not production page-table ownership. It does not replace Limine's
active mappings, enforce final kernel text/rodata/data permissions, reclaim
bootloader page tables, create user address spaces, or handle live page faults.

## Deliverables

- Safe bounded page-table arena with one mapper-issued typed root-table
  identity, a checked root-table identity path, and fixed table capacity.
- Unchecked page-table root reporting is test-only; external callers must use
  the audit-backed checked root path.
- Checked status reporting backed by the mapper audit path.
- Unchecked page-table status reporting is test-only; external callers must use
  the audit-backed checked status path.
- Kernel address-space candidate preflight combining audit, checked root,

- Canonical virtual-address validation.

## Security Notes

- Mapper construction rejects empty arenas.
- `root_table` is test-only and exposes a mapper-issued typed model root index
  only to crate tests; external code cannot construct a `PageTableRoot`
  directly, cannot read the internal model table index, and the value does not
  expose or claim a CR3-ready physical root frame.
- `root_table_checked` runs mapper audit before returning the opaque typed model
  root token without mutating mapper state, so corrupted root metadata is
  rejected before future activation code can consume it.
- `status_checked` runs the same corruption checks as `audit` before reporting
  total, used, and mapped page-table counters without mutating mapper state.
- `verify_kernel_address_space_candidate` returns an audit only after checked
  root identity, checked status accounting, non-empty mapping state,
  kernel-only mappings, and no-user-space, no-device, and no-physical-alias
  policy all pass. It is a read-only preflight for future activation work, not
  a live CR3 switch.
- Kernel candidate preflight performs address-space, user-privilege, and
  device-memory shape checks in one read-only mapping walk, with low-half
  address-space violations reported before flag-policy violations.
- `verify_user_address_space_candidate` returns an audit only after checked root
  identity, checked status accounting, non-empty mapping state, high-half
  user-access rejection, low-half kernel-privilege rejection, at least one
  low-half user mapping, no-device policy, no-global policy, and
  no-physical-alias policy all pass. It is a read-only preflight for future
  per-task address-space work, not a live CR3 switch.
- User candidate preflight performs address-space, privilege, required low-half
  user mapping, device-memory, and global-mapping checks in one read-only
  mapping walk, with missing user mappings reported before device/global
  policy violations.
- Candidate preflights run structural audit/root/status validation before
  non-empty and mapping-policy validation. They return `CorruptTable` for
  inconsistent mapper state even when the same mapper also contains
  policy-invalid mappings.
- Contiguous map/protect/unmap mutation paths reject accounting drift and
  malformed next-table links without committing candidate changes back into the
  original mapper.
- Standalone mapping policy helpers run a structural audit before evaluating
  policy predicates, so malformed or accounting-drifted tables report
  `CorruptTable` before user, executable, writable, device, global, or alias
  policy failures.
- Candidate preflights return `EmptyAddressSpace` for structurally valid
  mappers with no leaf mappings, so future activation code cannot accidentally
  consume an empty model root as a meaningful address space.
- User candidate preflight returns `IncompleteAddressSpace` for structurally
  valid mappers that contain mappings but no low-half user mapping, so a
  kernel-only mapper cannot be consumed as a future task address space.
- Mapping validates canonical virtual addresses and 4 KiB alignment before any
  state mutation.
- Mapping validates physical address range and 4 KiB alignment before any state
  mutation.
- Single-page map/protect/unmap operations structurally audit the mapper before
  slot or accounting mutation, so pre-existing corruption returns
  `CorruptTable` without extending, changing, or partially removing mappings.
- Device/cacheability flags are not public mutable state.
- Mapping entry encoding rejects executable device-memory mappings even if a
  caller bypasses the `device()` helper.
- Mapping entry encoding rejects user-global mappings even if a malformed
  in-crate flag value bypasses the public `with_global()` constructor.
- Capacity and mapping-flag checks are validate-then-commit, so out-of-table
  and invalid-flag failures leave the mapper unchanged.
- Contiguous range operations validate zero counts, endpoint arithmetic,
  canonical virtual-range boundaries, physical-range bounds, and requested flag
  encoding before walking or mutating a candidate mapper.
- Contiguous range operations reject ranges larger than the bounded mapper arena
  can walk, avoiding accidental unbounded scans in lookup or guard-page checks.
- Contiguous range map/protect/unmap operations run on a candidate mapper and
  commit only after every page succeeds, so range failures leave the
  caller-visible mapper unchanged.
- Mapping accounting drift and arithmetic checks fail closed before mutating
  page slots or reclamation state.
- Next-table slot encoding rejects unencodable table indices instead of
  truncating them into raw slot state.
- Existing mappings are not overwritten silently.
- `mapping_for_page` validates canonical page-aligned virtual addresses,
  structurally audits the mapper, and returns stored permissions without
  mutating mapper state.
- `is_page_mapped` validates canonical page-aligned virtual addresses,
  structurally audits through `mapping_for_page`, and reports presence without
  exposing physical addresses or mutating mapper state; corrupt intermediate
  table state and accounting drift are still reported as corruption.
- `ensure_mapped_contiguous` validates a non-empty virtual range, structurally
  audits the mapper, and verifies every page has a mapping without requiring
  physical contiguity, matching the future need to check required kernel
  regions without exposing frame details.
- `ensure_kernel_mapped_contiguous` validates a non-empty virtual range,
  structurally audits the mapper, and verifies every mapped page is
  kernel-privileged without requiring physical contiguity, so mixed address
  spaces can still validate protected kernel regions.
- `ensure_user_mapped_contiguous` validates a non-empty virtual range,
  structurally audits the mapper, and verifies every mapped page is
  user-privileged, giving future user address spaces a range-local check.
- `ensure_write_protected_contiguous` validates a non-empty virtual range,
  structurally audits the mapper, and rejects writable pages, giving future text
  and read-only data policy a range-local check.
- `ensure_non_executable_contiguous` validates a non-empty virtual range,
  structurally audits the mapper, and rejects executable pages, giving future
  data, stack, guard-adjacent, and device policy a range-local check.
- `ensure_executable_contiguous` validates a non-empty virtual range,
  structurally audits the mapper, and rejects non-executable pages, giving
  future text policy a range-local check.
- `ensure_normal_memory_contiguous` validates a non-empty virtual range,
  structurally audits the mapper, and rejects device-memory pages, giving future
  RAM-only regions a range-local check.
- `ensure_local_contiguous` validates a non-empty virtual range, structurally
  audits the mapper, and rejects global pages, giving future per-address-space
  mappings a local-TLB policy check.
- `ensure_kernel_space_contiguous` validates a non-empty high-half virtual
  range, structurally audits the mapper, and verifies every mapped page is
  kernel-privileged.
- `ensure_user_space_contiguous` validates a non-empty low-half virtual range,
  structurally audits the mapper, and verifies every mapped page is
  user-privileged.
- `mapping_for_contiguous` validates non-empty coherent page ranges and rejects
  gaps, physical discontinuity, mixed flags, or structural audit failures
  instead of reporting fragmented or corrupt mappings as a single range.
- Read-only range and range-policy checks return `CorruptTable` without
  mutating mapper state after malformed table links, intermediate leaves, or
  accounting drift are detected, including kernel/user privilege and
  address-space range policy variants.
- `ensure_unmapped_contiguous` validates non-empty page ranges and returns
  `AlreadyMapped` if any page in the range is mapped after a structural audit,
  giving guard-page and null-page policy a read-only check primitive.
- `visit_mappings` walks only reachable fourth-level leaf mappings, reconstructs
  canonical virtual page addresses, does not mutate mapper state, propagates
  callback errors, and rejects unsupported intermediate leaf slots, table
  cycles, duplicate table links, unreachable used tables, and non-empty unused
  tables.
- `mapping_summary` reuses the corruption-checked visitor and reports only
  aggregate page classes, giving policy code audit inputs without disclosing
  frame addresses; accounting drift and corrupt table shape return
  `CorruptTable` without mutating mapper state.
- Mapper-produced status, audit, and mapping-summary report debug output is
  aggregate-only and does not include raw slots, raw entry values, virtual
  addresses, or physical addresses.
- `PageTableMapper` implements a custom debug view with total/used/mapped
  counters and an audit-health boolean only. Mapping, visited-mapping, range,
  translation, flush-target, outcome, raw-entry, internal raw-slot, and
  validated-range debug output also redacts address-bearing fields instead of
  printing table arrays, slot raw values, virtual addresses, or physical
  addresses.
- `ensure_no_user_mappings` rejects any user mapping in a candidate kernel
  mapper while preserving corruption checks from the mapping visitor.
- `ensure_no_kernel_space_user_mappings` allows future low-half user address
  spaces while rejecting user-accessible mappings in the high-half kernel
  region.
- `ensure_no_user_space_kernel_mappings` allows future high-half kernel mappings
  while rejecting kernel-privileged mappings in the low-half user region.
- `ensure_no_executable_mappings` rejects any executable mapping in a candidate
  data-only mapper while preserving corruption checks from the mapping visitor.
- `ensure_no_writable_mappings` rejects any writable mapping in a candidate
  read-only mapper while preserving corruption checks from the mapping visitor.
- `ensure_no_device_mappings` rejects any device-memory mapping in a candidate
  normal-RAM mapper while preserving corruption checks from the mapping visitor.
- `ensure_no_global_mappings` rejects any global mapping in a candidate
  local-address-space mapper while preserving corruption checks from the
  mapping visitor.
- `ensure_no_physical_aliases` rejects duplicate physical-frame mappings in a
  candidate exclusive-ownership mapper while preserving corruption checks from
  the mapping visitor. This is an explicit policy gate; future shared-memory
  mappings should opt into a separate sharing model.
- `ensure_contiguous_flags` validates a virtual range and verifies expected page
  permissions without requiring physical contiguity, rejects invalid expected
  flags before auditing/walking, and gives v0.16 policy checks an audit-backed
  read-only permission primitive.
- `protect_page` validates canonical page-aligned virtual addresses, preserves
  the physical frame, rejects invalid flag combinations before mutation, and
  returns previous/current mappings plus a conservative flush target.
- Raw-slot encoding and decoding reject impossible writable+executable state,
  executable device-memory state, user-global state, partial cache-policy state,
  unknown leaf bits, empty slots, and next-table slots as mappings. Empty and
  well-formed next-table slots decode as non-leaf states; malformed non-empty
  slots, nonzero non-present slots, malformed next-table links, and decoded
  root-table child links return `CorruptTable` instead of being collapsed into
  an absent mapping.
- Raw x86_64 entry construction is limited to checked mapping encoding; callers
  cannot publicly construct arbitrary empty raw entries through the safe helper
  API.
- Leaf lookup, page-presence checks, protect, and unmap distinguish an absent
  leaf from malformed non-empty leaf state; malformed leaves return
  `CorruptTable` without mutating mapper state.
- `translate_checked` validates the virtual address, structurally audits the
  mapper, and reports invalid virtual addresses, unmapped pages, malformed
  leaves, accounting drift, and arithmetic overflow as explicit
  `PageTableError` variants. `translate` uses the same fail-closed `Result`
  shape instead of collapsing errors into an optional physical address.
- `translate_contiguous_range_checked` validates non-empty byte spans, rejects
  non-canonical or overlarge ranges before auditing/walking, and only returns
  one `TranslatedRange` when the mapper is structurally clean and every covered
  page is mapped, physically contiguous, and carries the same flags.
- Translate returns `None` for non-canonical or unmapped addresses.
- Unmap returns the removed mapping and a conservative TLB flush target.
- Unmap reclaims empty intermediate page tables while preserving tables that
  still contain sibling mappings.
- Global mapping removal or permission changes request an address-space flush
  instead of a page-only flush target.
- Merging two distinct page flushes conservatively promotes the result to an
  address-space flush instead of dropping either page.
- Normal boot prints counts and booleans only; it does not print raw virtual or
  physical addresses.

## Known Limitations

- The mapper is a model used by tests and smoke code; it does not activate CR3.
- Existing Limine-provided page tables remain active.
- No kernel section permission policy exists yet.
- No page-table frame allocation from the physical allocator exists yet.
- No address-space object exists yet.
- No user mappings or userspace page-fault recovery exist yet.
