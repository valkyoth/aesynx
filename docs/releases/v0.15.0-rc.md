# Aesynx v0.15.0 Release Candidate Notes

Status: implementation candidate; pending pentest and GitHub checks before tag.

## Scope

`v0.15.0` adds the first safe bounded page-table mapper model in `aesynx-mm`
and a normal-boot QEMU smoke that exercises map, translated-offset lookup,
unmap, and explicit TLB flush targets. The mapper is x86_64-shaped and uses
four 512-entry levels, but it does not activate a new hardware CR3 root yet.

This is not production page-table ownership. It does not replace Limine's
active mappings, enforce final kernel text/rodata/data permissions, reclaim
bootloader page tables, create user address spaces, or handle live page faults.

## Deliverables

- `aesynx-mm::PageTableMapper`.
- Safe bounded page-table arena with one root table and fixed table capacity.
- x86_64-shaped 4-level, 512-entry indexing for 4 KiB pages.
- Canonical virtual-address validation.
- Page-aligned virtual and physical mapping validation.
- Physical-address range validation for x86_64-style page-table entries.
- Map, translate, and unmap operations.
- Explicit `TlbFlush` target values for mapping updates.
- Double-map rejection.
- Atomic capacity failure behavior for map operations.
- `X86_64PageTableEntry` encoding for safe mapping flags.
- Fail-closed raw-slot decoding for impossible or non-leaf table state.
- Normal QEMU boot serial markers for page-table status and
  `[TEST] page-table=ok`.
- v0.15 QEMU image, manifest, serial-log names, help text, and marker tests.

Expected normal boot serial shape:

```text
[TEST] memory-map=ok
frame-allocator total_frames=<n> known_frames=<n> free_before=<n> free_after=<n> reserved_frames=<n> contiguous_count=2 double_free_check=true
[TEST] frame-allocator=ok
page-table total_tables=<n> used_tables=<n> mapped_before_unmap=1 mapped_after_unmap=0 translate_offset_ok=true flush_page=true
[TEST] page-table=ok
[TEST] bootinfo=ok
[TEST] boot=ok
```

## Security Notes

- Mapper construction rejects empty arenas.
- Mapping validates canonical virtual addresses and 4 KiB alignment before any
  state mutation.
- Mapping validates physical address range and 4 KiB alignment before any state
  mutation.
- Capacity checks are validate-then-commit, so out-of-table failures leave the
  mapper unchanged.
- Existing mappings are not overwritten silently.
- Raw-slot decoding rejects impossible writable+executable state, user-global
  state, empty slots, and next-table slots as mappings.
- Translate returns `None` for non-canonical or unmapped addresses.
- Unmap returns the removed mapping and a page-specific TLB flush target.
- Normal boot prints counts and booleans only; it does not print raw virtual or
  physical addresses.

## Known Limitations

- The mapper is a model used by tests and smoke code; it does not activate CR3.
- Existing Limine-provided page tables remain active.
- No kernel section permission policy exists yet.
- No page-table frame allocation from the physical allocator exists yet.
- No address-space object exists yet.
- No user mappings or userspace page-fault recovery exist yet.
- No page-table reclamation exists yet.
