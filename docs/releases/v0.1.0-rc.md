## Scope

`v0.1.0` is the repository foundation release. It does not claim a bootable
kernel, QEMU image, userspace runtime, hardware support, or production security.

The release includes:

- Rust workspace foundation.
- Stable Rust `1.96.0` toolchain pin.
- EUPL-1.2 license.
- Security policy.
- Threat model.
- Unsafe policy.
- Modularity policy.
- Kernel engineering policy.
- Release pentest gate.
- GitHub CodeQL default setup for Rust.
- Dependency, advisory, and source policy.
- Initial no-std workspace crates.
- `xtask` scaffold.

## Known Limitations

- No bootable image yet.
- No QEMU run target yet.
- No custom-target kernel binary yet.
- No hardware interaction.
- No userspace execution.
- No external dependency exceptions currently exist.
- `v0.2.0` build skeleton work already exists on `main`, so the `v0.1.0`
  report describes the foundation criteria rather than an isolated branch.
- No real interrupt-control, core-ID, or timestamp implementation yet; arch
  stubs return explicit unsupported errors.
- No bytecode verifier yet.
- No IPC replay window yet.
- No concrete capability revocation epoch backing store yet.
- No concrete capability audit log backend yet.
- Scheduler priority bands are not separated yet.

## Static Pentest Follow-Up

The following `v0.1.0`-appropriate static-analysis findings were fixed before
the release candidate was considered tag-blocked:

- Page flags no longer permit writable+executable access.
- Page privilege is explicit instead of a raw user-accessible boolean.
- Architecture stubs no longer silently claim interrupt, core-ID, or timestamp
  support.
- Capability permissions reject undefined bits at trust boundaries.
- Capability grant requires `GRANT`.
- Capability live checks expose generation/epoch validation.
- Capability revocation epoch stores enforce `REVOKE` before epoch increment.
- Capability derive/grant call paths have audited variants and a required audit
  hook before authenticated call paths land.
- Inline IPC payloads are bounded to 64 bytes by construction.
- Secret scanning now includes TOML and YAML files.
- AI scheduling metrics use unsigned types.
- `cargo deny` unmaintained and unsound advisory policies are strict for all
  dependencies.
- Bytecode permission checks use `CapPerms` instead of raw `u32`.
- Bytecode fuel has a maximum.
- Log messages are bounded and reject embedded record separators.
- Boot logging uses an explicit rejected-message fallback instead of silently
  dropping rejected log records.
- Telemetry counters are private and exposed through snapshots.
- Release tag validation rejects path traversal.
- `xtask` resolves scripts from the workspace root.
- Kernel image layout is no longer carried in a `Copy` struct.
- CodeQL default setup is enabled in GitHub code scanning. No custom CodeQL
  workflow is committed, to avoid mixing default and advanced configurations.
