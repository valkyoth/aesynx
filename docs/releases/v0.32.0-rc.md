# Aesynx v0.32.0 Release Candidate Notes

`v0.32.0` prepares Aesynx for future AI-assisted policy without loading,
executing, or trusting an AI model.

## What Changed

- Split `aesynx-ai-policy` into focused no_std modules for confidence, policy
  decisions, model manifests, and scheduler features.
- Added schema-v1 `ModelObjectManifest` with model ID, kind, policy domain,
  input/output/weights hashes, signature, and safety limits.
- Added nonzero `Hash256` and `Signature64` wrappers so empty model metadata is
  rejected before a model can be admitted; this is explicitly a metadata
  presence gate, not cryptographic verification.
- Added redacted `Debug` output for manifest hashes, signatures, validated
  manifests, and model identity.
- Added scheduler-domain manifest gates: required fallback, bounded evaluation
  steps, bounded memory, bounded confidence, and required telemetry fields.
- Added `PolicyDecision::from_model` so model-backed decisions enforce
  `ModelSafetyLimits::max_confidence` by switching to deterministic fallback
  with `DecisionReason::SafetyRejected` when the model exceeds its declared
  ceiling.
- Added fixed-point scheduler feature validation with `FIXED_POINT_SCALE` across
  every non-priority scheduler input.
- Documented `ScheduleFeatures::priority` as a discrete rank rather than a
  fixed-point ratio.
- Added `VirtAddr::new_x86_64_canonical` as a shared ABI helper for future
  syscall/IPC boundary validation.
- Hardened BootInfo normalization so the kernel image physical footprint must
  be page-aligned, fully covered by `Kernel` or `Reserved` memory-map regions,
  and never overlap `Usable` memory before the memory map can seed a future
  frame allocator.
- Added an intentional `BootInfo::framebuffer_base()` accessor while keeping
  framebuffer base fields private and redacted in debug output.
- Hardened early x86_64 GDT/TSS and IDT setup with three-state
  uninitialized/in-progress/ready guards so re-entrant initialization cannot
  observe a table as ready before it is installed.
- Cleared FS/GS/KERNEL_GS base MSRs through an explicit admitted-MSR list when
  descriptor reload resets FS/GS selectors, avoiding residual bootloader or
  firmware segment-base state before future TLS/per-CPU setup.
- Made interrupt-gate installation use a release/debug-identical
  mask-and-restore error path when callers violate the "interrupts already
  masked" contract.
- Added `ObjectBoundedRange` so bounded capability derivation requests must
  carry a range already checked against the object's real backing extent.
- Added audited root minting through `CapabilityTable::insert_root_with_audit`
  and extended the kernel capability smoke to prove mint/derive/grant/revoke
  audit coverage.
- Tightened `Capability` debug output so it redacts permission bits,
  generation, and revocation epoch in addition to authority identifiers.
- Made device `Revoked` and `Crashed` states terminal sinks instead of allowing
  churn between terminal incident states.
- Hardened kernel heap deallocation so production `GlobalAlloc::dealloc`
  fail-stops on detected invalid free, double free, or corrupt free-list
  states instead of silently swallowing the result.
- Made slab-page reclaim validate the free-list chain before relinking retained
  free blocks, preserving fail-closed behavior if corruption is detected.
- Changed service-queue pending inspection for unsupported service kinds to
  return `UnsupportedService` instead of reporting `0` pending requests.
- Removed implicit `Copy` from `PageTableMapper` so large page-table arenas
  cannot be accidentally copied onto kernel stacks.
- Reworked contiguous page-table map/protect/unmap helpers to mutate in place
  with bounded prefix rollback instead of cloning the full mapper arena.
- Replaced page-table audit fixed-point rescanning with a bounded top-down
  traversal from the root table plus a validation/counting pass.
- Simplified empty-table reclaim so the production `level == 0` guard is the
  documented root-table protection instead of a redundant debug assertion.
- Redacted `aesynx-rt::Env` debug output so capability identifiers cannot leak
  through accidental runtime environment formatting.
- Made local executor dispatch/yield/sleep/wake counter-overflow paths reject
  before moving tasks or mutating queues.
- Changed executor restore-failure errors to preserve the rejected `Task`
  instead of dropping it while reporting only the queue error.
- Added a kernel AI-policy smoke that accepts a safe scheduler manifest, rejects
  a manifest without required fallback, and proves deterministic local fallback.
- Updated QEMU image names and manifest metadata to `v0.32.0`.

## Security And Correctness

- AI policy remains advisory only. The kernel enforces deterministic policy and
  capabilities; AI cannot grant authority.
- Model manifests are data, not executable code. Validation returns an opaque
  `ValidatedModelManifest` rather than trusting raw metadata directly.
- `ValidatedModelManifest` now documents that it means structural and policy
  validation only. It does not imply cryptographic verification of model hashes
  or signatures.
- Only fixed-point heuristic/table manifests are accepted in v0.32.0.
  `WasmComponent` and `NeuralNetwork` kinds are declared for the roadmap but
  rejected until real backends and sandboxes exist.
- Scheduler manifests must match the scheduler domain and include the minimum
  scheduler telemetry feature set.
- Deterministic fallback decisions have no model identity, zero confidence, and
  `DecisionReason::DeterministicFallback`.
- The QEMU smoke emits only bounded booleans, schema, local core zero, and
  fallback confidence zero before `[TEST] ai-policy=ok`.
- Actual model loading, inference runtimes, online learning, authority-bearing
  AI decisions, and model-object persistence remain future work.
- The QEMU smoke marker is `manifest_metadata_gate_ok=true`, deliberately not a
  cryptographic safety marker.
- Presence-only hashes/signatures remain barred from becoming a model execution
  trust boundary; a future model loader must add real signature/hash
  verification before admitting model weights.
- `ModelObjectManifest` equality is documented as structural/test convenience,
  not a constant-time authentication primitive.
- Spectre-class controls such as IBRS/STIBP/SSBD/ARCH_CAPABILITIES remain
  planned CPU-hardening work and are not claimed active in this release.
- Unaudited `CapabilityTable::insert_root` remains a bootstrap-only path for
  authority seeding before an audit sink exists; runtime object creation must
  use the audited mint path.
- Table revocation intentionally remains total per object in the current model.
  Audit events expose `affected_slots` so unexpectedly broad revokes can be
  detected before future per-grant revocation exists.
- Security-sensitive consumers of `require_random_tokens()` must refuse the
  operation or fail boot until a runtime self-test-backed entropy path exists;
  deterministic fallback remains anti-confusion only.
- Limine memory-map `entry_count` and the entries-array extent remain an
  accepted bootloader trust boundary: Aesynx bounds the copied count and
  validates each entry pointer after reading it, but cannot independently prove
  the bootloader's array allocation length from inside the kernel.
- Object registry generation exhaustion remains a deliberate fail-closed
  retirement signal. Before registries become long-lived multi-tenant
  resources, diagnostics must expose generation-exhaustion events so operators
  can detect capacity erosion.
- Address-bearing kernel types are not treated as cryptographic secrets in
  v0.32.0. Future per-domain address-space roots should revisit zeroization for
  high-value root-frame metadata if the threat model requires stale stack or
  register-spill minimization.

## Known Limitations

- No AI model is loaded or evaluated.
- No model object is stored in the object registry yet.
- Signature verification is represented by nonzero manifest metadata only; real
  cryptographic verification belongs to future package/model loading work.
- The scheduler still uses deterministic round-robin. v0.33.0 owns the
  non-AI scheduler policy model prototype.
