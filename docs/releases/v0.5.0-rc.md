# Aesynx v0.5.0 Release Candidate Notes

Status: implementation candidate; not tag-ready until pentest and GitHub checks
are complete.

## Scope

`v0.5.0` is the BootInfo-normalization milestone. It keeps Limine as the boot
path, but moves the kernel from fixed serial output to real bootloader handoff
metadata normalization.

It does not claim page-table ownership, bootloader memory reclamation, heap
allocation, interrupts, SMP topology discovery, module loading, or framebuffer
output. Those start in later milestones.

## Deliverables

- Dependency-free `aesynx-boot` normalized metadata types.
- Safe `BootInfo::normalize` validation path.
- Memory-map summary support.
- Limine request statics for memory map, executable address, HHDM,
  framebuffer, and RSDP metadata.
- `kaslr: yes` in the QEMU Limine config.
- Kernel serial summary from normalized BootInfo.
- QEMU boot smoke requiring both `[TEST] bootinfo=ok` and `[TEST] boot=ok`.
- Documented unsafe boundary for Limine handoff parsing.

Expected serial:

```text
Aesynx: booting
arch=x86_64 platform=qemu
memmap regions=<n> usable=<n> usable_bytes=<n>
rsdp=present
[TEST] bootinfo=ok
[TEST] boot=ok
```

## Security Notes

- `aesynx-boot` remains `#![no_std]`, `#![deny(unsafe_code)]`, and dependency
  free.
- Raw Limine response parsing is isolated to
  `crates/aesynx-kernel/src/limine.rs`.
- Limine memory-map entries are copied into a bounded early stack buffer before
  the generic `BootInfo` is constructed.
- `KernelImageInfo` remains debug-redacted and does not expose public address
  getters.
- QEMU KASLR is enabled now that executable-address metadata is consumed.
- The boot smoke checks the BootInfo marker so CI proves normalization ran.

## Verification

Required before pentest handoff:

```bash
scripts/checks.sh
scripts/generate-sbom.sh
cargo xtask build-kernel
cargo xtask image
cargo xtask qemu
```

## Known Limitations

- Bootloader response data is consumed only during early boot and is not
  reclaimed.
- Memory-map data is bounded by `MAX_EARLY_MEMORY_REGIONS`.
- CPU topology remains empty for v0.5.
- Framebuffer metadata is normalized but not rendered to.
- RSDP is recorded as metadata only; ACPI parsing starts later.
- `X86_64::halt_forever()` still uses a spin loop rather than `hlt`.
